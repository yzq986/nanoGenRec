# FuXi-Linear: Unleashing the Power of Linear Attention in Long-term Time-aware Sequential Recommendation

- **Arxiv**: 2602.23671
- **Authors**: Yufei Ye, Wei Guo, Hao Wang, et al. (USTC + Huawei Technologies)
- **Date**: 2026-02
- **Read date**: 2026-04-22

## Core Contribution
- 提出 FuXi-Linear：三通道线性 attention（Retention + Linear Positional + Temporal Retention），O(n) 训练复杂度 + O(1) 推理复杂度，适合数千 token 长序列
- Temporal Retention Channel：将时间信号用复数域建模（r·e^(iθΔt)），把 temporal query/key 从 semantic attention 中解耦，避免互相干扰，支持周期性建模
- Linear Positional Channel：用可学习 kernel 函数近似 RPE（相对位置编码），维持线性复杂度；比 RoPE 在推荐任务中更强

## Method
三通道并行处理后拼接 + gating，MFFN 两级 FFN 融合。支持 parallel/recurrent/chunkwise recurrent 三种计算形式。Temporal channel 用 H_t=8 组参数对 (r_h, θ_h) 多尺度建模，θ_k = 2π/(B^(b0+k)) 保证周期性。

## Key Results
在 MovieLens-20M、Kuairand-27K（avg seq len=3555）、KuaiRec（avg=1745）三个长序列数据集上：
- 相比 HSTU (SOTA softmax attention)：HR@10/NDCG 持平或略优，同时推理 10-21× 加速
- 相比其他线性 attention（Mamba4Rec, TTT4Rec, TiM4Rec）：明显优于所有线性基线
- 验证了在 1000+ 长度下仍有 power-law scaling

## Relevance to gr_demo
**低相关（短期）——针对 user 历史序列建模，非 SID 生成的 autoregressive decoder**：

1. **应用场景不同**：FuXi-Linear 替代的是历史序列 encoder 中的 softmax attention，而我们的 NTP decoder 处理的是短 SID 序列（最多 30 items × 3 tokens = 90 tokens）。目前不是瓶颈
2. **Temporal Retention Channel 的思路**：将时间信号从语义信号中解耦（独立的 Q/K 计算）是一个比 IDEA-feat-0（time gap embedding 直接相加）更精细的设计。若未来需要更长历史序列，可以参考
3. **周期性时间建模**：复数域建模 r·e^(iθΔt) 自然支持日/周/月周期性，与 IDEA-feat-5（TO-RoPE Time+Order Encoding）方向一致
4. **当前无直接实验价值**：我们的 user 历史序列是固定 30 items 窗口，线性 attention 无优势。若扩展到长序列（>200 items），FuXi-Linear 是参考候选
