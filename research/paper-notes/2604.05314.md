# Next-Scale Generative Reranking: A Tree-based Generative Rerank Method at Meituan

- **Arxiv**: 2604.05314
- **Authors**: Shuli Wang, Changhao Li, Ke Fan, Senjie Kou, Junwei Yin, Chi Wang, Yinhua Zhu, Haitao Wang, Xingxing Wang (Meituan)
- **Date**: 2026-04
- **Read date**: 2026-04-22

## Core Contribution
- Propose a **next-scale generation paradigm** for reranking: a tree-based generator (NSG) that progressively expands a recommendation list in a coarse-to-fine "one-generates-two, two-generate-four" binary-tree pattern, balancing global and local perspectives — unlike autoregressive (local-only), one-step (global-only), or multi-step (local-optima-prone) methods.
- Design a **multi-scale neighbor loss (MSNL)** that leverages a tree-based multi-scale evaluator (MSE) to provide scale-specific guidance at each tree level, addressing the goal inconsistency problem between generator and evaluator using relative rewards inspired by GRPO.
- Demonstrate state-of-the-art on both Taobao Ad and Meituan industrial datasets, with successful **online deployment** on Meituan food delivery: +2.89% CTR, +3.15% GMV vs YOLOR baseline when scaling from 8 to 20 candidate items.

## Method
NSGR operates in two modules. The **Multi-Scale Evaluator (MSE)** extracts user interest via HSTU on lifelong behavior sequences (items represented as 3-level semantic IDs), enhances candidate item embeddings via MLP fusion with user interest and short-term behavior (target attention), then computes multi-scale context via self-attention at K=log₂(m) granularities (full list → pairs) — aggregated into a position-aware CTR prediction. The **Next-Scale Generator (NSG)** starts from the full candidate set and recursively performs binary splits at each scale: it computes item priority scores, classifies pairwise relationships (suppression/enhancement/neutral), refines item representations via set-conditioned interactions, then scores each item's alignment with an anchor vector (self-attention over user interest + tree history) to split the set into two halves. This recurses for K=log₂(m) steps, yielding the final ranked list. Training uses frozen MSE as reward model: neighbor lists are constructed by swapping items at each tree node, MSE evaluates their multi-scale representations, and a contrastive loss (MSNL, Eq. 22) pulls the generator toward higher-valued neighbors and pushes away from lower-valued ones at each scale. HSTU is pretrained with NTP.

## Key Results

**Offline (Meituan dataset, A²⁰₂₀ permutation space):**

| Model | AUC | GAUC | Loss |
|-------|-----|------|------|
| PRM | 0.8595 | 0.8573 | 0.1008 |
| NLGR | 0.8732 | 0.8644 | 0.0946 |
| YOLOR | 0.8749 | 0.8669 | 0.0932 |
| **NSGR** | **0.8902** | **0.8829** | **0.0842** |

NSGR: +0.0153 AUC, +0.0160 GAUC over best baseline.

**Generator consistency (Meituan):** HR@1% = 0.861, HR@10% = 0.987 (vs YOLOR 0.822/0.943).

**Online A/B (8 weeks, Meituan food delivery, vs YOLOR-8):**

| Method | CTR | CVR | GMV | Latency |
|--------|-----|-----|-----|---------|
| NSGR(8) | -0.42% | -0.18% | -1.02% | -2.1ms |
| NSGR(20) | **+2.89%** | +0.58% | **+3.15%** | -1.4ms |

Key insight: NSGR(8) slightly underperforms YOLOR(8) which can exhaustively enumerate A⁸₈=40320 permutations; NSGR's advantage materializes when the permutation space is large (A²⁰₂₀=2.43×10¹⁸) where exhaustive evaluation is infeasible.

**Ablation (Meituan):** Removing SID→ AUC drops to 0.8761; removing MSE unit→ 0.8835; removing NSG unit→ HR@1% drops to 0.796; removing MSNL→ HR@1% drops to 0.772.

## Relevance to nanoGenRec
nanoGenRec uses hierarchical SID tokens (L0→L1→L2) via RQ/MLP-FSQ tokenizer, NTP transformer prediction, and beam search decoding (baseline PPL=25.94, R@500=61.2%). NSGR's relevance is **indirect but architecturally informative**:

1. **Coarse-to-fine generation matches our SID hierarchy**: NSGR's tree-based "next-scale" generation (progressively refining from global user interest to fine-grained positions) is conceptually parallel to our L0→L1→L2 SID generation. The key insight is that **each SID level should receive scale-specific supervision** — not just NTP cross-entropy at each token. NSGR's multi-scale neighbor loss, which uses evaluator feedback at each tree level, suggests we could train a lightweight evaluator that provides per-level reward signals to guide beam search or RL training. This is complementary to IDEA-promise-0 (PRM-guided beam search).

2. **Multi-scale evaluator as reranker for beam candidates**: NSGR's MSE computes multi-scale context via self-attention at different granularities. For nanoGenRec, after beam search produces top-K SID candidates, a multi-scale reranker could evaluate candidates using context at L0 (coarse cluster), L1 (sub-cluster), and L2 (leaf) levels simultaneously — rather than relying solely on the NTP model's joint probability. This is a lightweight post-beam reranking step.

3. **Pairwise item relationships in candidate set**: NSGR's NSG unit models competitive suppression vs complementary enhancement between items. For nanoGenRec's beam search, this suggests a **diversity-aware beam reranking** step: among the top-K beam candidates, suppress redundant items (same L0/L1 prefix) and boost complementary ones, potentially improving R@500 by expanding coverage.

4. **Neighbor loss for RL alignment**: NSGR's multi-scale neighbor loss (constructing counterfactual lists by swapping items and using relative evaluator scores as reward) is directly applicable to our planned GRPO/DPO training. Instead of generic positive/negative pairs, construct neighbors by swapping individual SID tokens at each level and use a reward model to evaluate the resulting item relevance.

**Expected gain magnitude**: NSGR's gains are in the reranking stage (CTR/GAUC), not retrieval recall. Direct application of the full NSGR framework is not applicable (we do retrieval, not reranking). However, the multi-scale evaluator as a beam reranker and neighbor loss for RL could each contribute **+1-3% relative R@500 improvement** based on analogous gains from PRM (IDEA-promise-0) and contrastive RL approaches.
