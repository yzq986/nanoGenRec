# Surgical Post-Training (SPOT): Cutting Errors, Keeping Knowledge

[English](2603.01683.md) | [Chinese](2603.01683.zh.md)

- **Arxiv**: 2603.01683
- **Authors**: Wenye Lin, Kai Han (University of Hong Kong)
- **Date**: 2026-03
- **Read date**: 2026-04-22

## Core Contribution
- 揭示了 DPO implicit reward 的"Elastic Tether"效应：KL 约束通过动态梯度系数 λ = 1 - σ(r_θ) 实现自适应正则，防止 catastrophic forgetting，而非 on-policy 数据本身
- 提出外科手术式数据整流管道：Oracle 对模型错误步骤做 minimal edit（用 LCS 约束 RLCS < 0.6），生成贴近模型分布的 (y-, y+) 对
- SPoT-BCO 目标：解耦 DPO 为独立二分类任务，用 BCO（adaptive reward shift δ 防止梯度消失），比 DPO 更适合 rigid correctness 的推理任务；仅 4k 样本+28 分钟训练，Qwen3-8B 平均提升 +6.2%

## Method
**数据管道**：对模型失败样本 y- 用 Oracle（Gemini 2.5 Pro）做最小化修正得 y+，要求 RLCS < 0.6（y+/y- 的 LCS 覆盖率 > 40%）。
**损失函数**：
- SPoT-BCE: L = -E[log σ(r(x,y+)) + log σ(-r(x,y-))]（解耦 DPO）
- SPoT-BCO: L = -E[log σ(r(x,y+)-δ) + log σ(-(r(x,y-)-δ))]，δ 是 batch 统计的指数移动平均

**理论支撑**：SFT 梯度系数恒为 1；Reward-SFT 梯度系数为 λ = 1-σ(r_θ)，r_θ 大时趋零（"饱和模式"），防止过度优化，自动止步。

## Key Results
Qwen3-8B+SPoT-BCO vs Qwen3-8B:
- In-domain avg: 46.8% → 52.1% (+5.3%)
- OOD avg: 29.9% → 41.4% (+11.5%)
- IFEval: 83.0% → 84.8% (+1.8%)

DPO 能防 forgetting 但不提升 in-domain 推理；SFT 提升 in-domain 但严重 forgetting；SPoT-BCO 两者兼顾。

## Relevance to nanoGenRec
**高相关——直接影响我们的 DPO/对齐训练设计**：

1. **现有 SP-DPO 的改进方向**：我们的 SP-DPO 使用标准 DPO loss。SPOT 指出 DPO 倾向于通过打压 y- reward 而非提升 y+ reward 来最小化 margin——这在生成推荐中对应"降低 beam 内 bad SID 的概率"而非"提升 target SID 的概率"。改为 SPoT-BCE/BCO 可能更有效
2. **BCO 中的 δ shift**：对于 SID 预测，target SID beam 的 reward 可能不在同一量级，adaptive δ 防止梯度消失对长 SID token 序列尤为重要
3. **数据整流管道**：在 SP-DPO 场景，"beam 搜索产生的 wrong SID" = y-，"teacher-forced target SID" = y+。LCS 约束可以控制数据质量：only include samples where beam partially matches target（RLCS < threshold），确保梯度集中在分歧 token
4. **Elastic Tether 理解**：解释了为何我们的 SP-DPO 相比 NTP baseline 有时不稳定——可能是 DPO 过度抑制了 rejected beam 而非提升了 target SID
5. **具体实验方向（IDEA-spot-0）**：
   - 用 SPoT-BCO 替换 SP-DPO 的 loss function
   - 对 sp-dpo-prepare 的 beam pairs 加 LCS 质量过滤（target 和 best beam 的 SID token 序列 LCS > 30%）
   - 无需重训 NTP，只改 alignment-eval 阶段
