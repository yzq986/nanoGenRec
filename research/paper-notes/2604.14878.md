# GenRec: A Preference-Oriented Generative Framework for Large-Scale Recommendation

- **Arxiv**: 2604.14878
- **Authors**: Yanyan Zou, Junbo Qi, Lunsong Huang, Yu Li, Kewei Xu, Jiahao Gao, Binglei Zhao, Xuanhua Yang, Sulong Xu, Shengjie Li (JD.com / Waseda University)
- **Date**: 2026-04
- **Read date**: 2026-04-22

## Core Contribution
- **Page-wise NTP (PW-NTP)**: Replaces vanilla point-wise NTP with page-wise supervision — target sequence is the full page of interacted items (ordered ∪ clicked ∪ exposed), resolving the one-to-many ambiguity where identical inputs map to multiple valid labels. Hallucination rate drops >50%, converges faster, denser gradient signal per forward pass.
- **Token Merger**: Asymmetric linear projection compresses 3-token SID triplets into a single latent vector in the prefilling/prompt side via `h = Linear(Concat(e(s1), e(s2), e(s3)))`, reducing input length ~2× with negligible accuracy loss (HR@50: 0.7192 vs 0.7201 w/o TM). Decoding side remains unmerged for full-resolution generation.
- **GRPO-SR**: Group Relative Policy Optimization with Supervised Regularization. Adds NLL regularization on positive trajectories (replacing KL divergence), plus a Hybrid Reward system: dense SIM-based reward gated by relevance threshold G=I(sim>τ) to prevent reward hacking, and positive-item calibration (force actual clicks/purchases to group max reward). Reward@1 improves +18.01% over SFT baseline.

## Method
GenRec is a decoder-only generative retrieval framework (Qwen2.5 backbone, 3B main variant) for large-scale recommendation deployed on JD.com. Items are tokenized into 3-level Semantic IDs via multimodal Qwen2.5-VL encoding → collaborative fine-tuning → RQ K-Means quantization. Training is two-stage: (1) Page-wise NTP SFT where the model autoregressively predicts an entire page of items (ordered by interaction intensity: purchase > click > expose) rather than a single item, with an asymmetric Token Merger compressing multi-token SID inputs in the prompt side while keeping full-resolution decoding; (2) GRPO-SR reinforcement learning where the model generates single-item rollouts (matching beam search inference protocol), scored by a hybrid reward combining a dense SIM-based preference model with a relevance gate to suppress reward hacking, and an NLL regularizer on positive samples to anchor the policy to real user behavior. Inference uses standard point-wise beam search.

## Relevance to nanoGenRec
Our system uses hierarchical SID tokens L0→L1→L2 via RQ/MLP-FSQ tokenizer, NTP transformer prediction, beam search decoding. Current baseline: PPL=25.94, R@500=61.2%.

**1. Page-wise NTP (IDEA-genrec-0) — HIGH RELEVANCE**:
- Our current training uses point-wise NTP (one target item per input). If our dataset contains multiple positive interactions per session/page (same user context), PW-NTP directly resolves the one-to-many ambiguity.
- GenRec reports: PW-NTP vs vanilla NTP (same LC-Rec architecture) → HR@50 0.7192 vs 0.6226 (+15.5%), HaR 4.96% vs 7.80% (hallucination halved).
- **Expected gain for nanoGenRec**: If we have multi-positive sessions, this could improve R@500 by 5-10% and reduce invalid SID generation. Training converges faster (denser signal per forward pass). Requires data pipeline change to group interactions by session/page.
- **Implementation**: Change target from single `[L0, L1, L2]` to concatenated `[L0_1, L1_1, L2_1, L0_2, L1_2, L2_2, ...]` for all positive items in a session, ordered by interaction strength. Keep beam search inference unchanged.

**2. Token Merger (IDEA-genrec-1) — MODERATE RELEVANCE**:
- Our system uses 3-token SIDs (L0, L1, L2). Token Merger compresses these 3 tokens per item to 1 token in the prompt, reducing input length ~2× with <0.1% accuracy loss.
- **Expected gain**: Not accuracy improvement but ~2× prefilling speedup and ability to fit longer user histories within the same context window. At our current sequence lengths this may not be critical, but becomes important when scaling to longer histories (>100 items × 3 tokens = 300+ tokens).
- **Implementation**: Add `nn.Linear(3*embed_dim, embed_dim)` applied to concatenated SID embeddings in the encoder/prompt side. Keep decoding unmerged.

**3. GRPO-SR (IDEA-genrec-2) — HIGH RELEVANCE (post-SFT)**:
- NLL regularization (anchoring to real behavior) is simpler than KL divergence (no reference model inference needed) and directly applicable.
- Relevance gate G is critical: prevents reward hacking where model generates valid-but-irrelevant SID combinations.
- Positive calibration (forcing actual positives to max group reward) corrects reward model bias — important when using proxy rewards.
- **Expected gain**: GenRec shows +18% Reward@1, +3.4% HR@50 over SFT baseline. For nanoGenRec, RL alignment after SFT could add 2-5% absolute R@500.
- **Prerequisite**: Need solid SFT baseline first (current PPL=25.94 is reasonable). Need reward model or proxy reward signal.

**4. Scaling insight**: Depth > width for GR — 3B (36L, 2048d) ≈ 7B (28L, 3584d). Our current 6-layer model may benefit more from adding layers than widening.
