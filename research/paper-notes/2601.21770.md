# OneMall: One Architecture, More Scenarios — End-to-End Generative Recommender Family at Kuaishou E-Commerce

[English](2601.21770.md) | [Chinese](2601.21770.zh.md)

- **Arxiv**: 2601.21770
- **Authors**: Kun Zhang, Jingming Zhang, Wei Cheng, et al. (Kuaishou Technology)
- **Date**: 2026-01
- **Read date**: 2026-04-22

## Core Contribution
- **ResKmeansFSQ Tokenizer**：前 2 层用 Res-Kmeans（语义保真），最后 1 层用 FSQ（Binary 16-bit MLP，4096 code）确保唯一性，SID 冲突率从 36% 降至 11%，HR@500 +4.1pp
- **Decoder-Only + Query-Former + Sparse MoE**：多行为序列（Exposure/Click/Buy）各自用 Query-Former 压缩成 4×D 向量，cross-attention 融合，Sparse MoE（24专家/2激活，loss-free balancing）做 autoregressive generation；in-batch contrastive 辅助目标（s³_L 与 item embedding，98% acc@1）
- **GRPO > DPO for RL**：在电商场景（Top10/100/500 所有 beam 大小）GRPO 均优于 DPO，原因是 GRPO 在全组 768 候选上归一化 advantage，提供更丰富梯度信号

## Method
**Tokenizer**：LLM 微调（Qwen2.5 1.5B + Swin-ViT）生成 item embedding，Res-Kmeans 2层 + FSQ 1层（binary 16-bit MLP）。Live-streaming 用 two-tower item tower embedding + 当前售卖商品 SID 作为附加特征。

**Architecture**：
- Query-Former (M=10 query tokens per sequence) 压缩长序列（500→10），节省 3.7× FLOPs
- Cross-attention：BOS+SID 序列对历史序列做 cross-attn
- Causal Self-attention：SID 序列内部的 causal mask
- Sparse MoE FFN：DeepSeek V3 style，loss-free load balancing

**RL**：Reference model 周期同步，beam 768 candidates，ranking model 打分，F1 fusion reward，GRPO/DPO 对比，最终用 GRPO，2% 训练数据做 RL。

## Key Results
**Scaling** (Short-Video, HR@500): 0.05B→0.6, 0.1B→0.632, 0.5B-A0.1B(MoE)→0.742, 1.3B-A0.1B→0.760
MoE 从 dense 0.1B → 0.5B-A0.1B 的 jump 最大（+11% HR@50），验证 sparse MoE 的有效性。

**RL Ablation**: GRPO > DPO > Base in Top10/100/500 all beam sizes.

**Online**: +14.71% GMV (Product-Card), +10.33% GMV (Short-Video), +4.90% GMV (Live-Streaming)

**Auxiliary Contrastive Loss**: +1.5%/+1.7%/+1.7% on HR@50/100/500

## Relevance to nanoGenRec
**高相关——与我们架构最接近的工业实践，多处直接可借鉴**：

1. **ResKmeansFSQ 冲突率问题**：我们当前用 MLP-FSQ 32-bit（3层），OneMall 的 2层 Res-Kmeans + 1层 FSQ 与我们架构非常接近。他们的 36%→11% 冲突率改善，提示可能值得检验我们自己的冲突率
2. **Query-Former 序列压缩**：如果未来扩展多行为序列（click + buy + expose），Query-Former 是低成本压缩方案（10 query tokens per seq）。当前我们只用单一序列，不紧急
3. **in-batch Contrastive Loss 作为辅助**（IDEA-onemall-0 重新审视）：OneMall 报告辅助 contrastive loss +1.7% HR@500。但我们 EXP-022 测试是 negative。关键区别：OneMall 在已有 ResKmeansFSQ 基础上加 contrastive，而我们在 FSQ-only tokenizer 上加——说明 contrastive 需要更好的 item representation 作为基础
4. **GRPO 在 GR 场景验证**：第 3 个独立验证 GRPO > DPO（OneLive 2 + OneMall）。强烈支持探索 GRPO 而非仅 DPO
5. **Scaling 数据**：0.5B-A0.1B（MoE）比 dense 0.1B 高 +11% HR@50。我们的 scaling law 拟合（S=17M vs M+=101M）结果对应，但分析维度不同（loss 而非 HR）
