# TO-RoPE: Time-and-Order RoPE for Generative Recommendation

[English](2510.20455.md) | [中文](2510.20455.zh.md)

- **Arxiv**: 2510.20455
- **Authors**: Xiaokai Wei, Jiajun Wu, Daiyao Yi, Reza Shirkavand, Michelle Gong (Roblox + UMD)
- **Date**: 2025-10
- **Read date**: 2026-04-22

## Core Contribution
- **TO-RoPE**: 统一框架，将时间和 index 作为 rotary embedding 的双角度来源：θ_k(i) = (1-λ_k)·α^p_k·i·ω^p_k + λ_k·α^t_k·τ_i·ω^t_k
- **三种实例化**：
  - Early Fusion：每个 plane 内角度直接相加（表达力强但有相位干扰）
  - Split-by-Dimension：每个 plane 单独选择 index 或 time（无 plane 内干扰，容量可调）
  - Split-by-Head：每个 head 专化为 index-only 或 time-only（head 级别隔离，细粒度频率覆盖）
- **时间的好处**：RoPE 的 frequency ladder 自然建模 bursts（高频）、长期 rhythms（低频）、周期性（特定频率 ≈ day/week/season），优于手工 time gap 特征

## Method
时间归一化：τ_i = (u_i - u_ref) / s。Wavelength-targeted 构建频率库：ω_max = 2π/λ_min, ω_min = 2π/λ_max，log-space 填充。Split-by-dim 推荐 50% time / 50% index（实验显示 30%-50% time 为 safe default）。与 FlashAttention 兼容（只改 Q/K rotation，不改 attention 核）。

## Key Results
工业数据集（游戏平台，tens of millions users）：
- Index APE: HR@10=0.5493, TO-RoPE split-by-dim/head: HR@10=0.5582 (+1.6%)
- Best: split-by-head (0.5582) ≈ split-by-dim (0.5582) > Early Fusion (0.5562) > Time-only RoPE (0.5568) > Index-only (0.5537) > HSTU-style relative bias (0.5513)

ML-20M：Split-by-dim HR@10=0.3406 vs HSTU relative bias=0.3341 (+2.0%)
