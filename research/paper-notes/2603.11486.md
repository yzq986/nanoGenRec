# Quantized Inference for OneRec-V2

[English](2603.11486.md) | [Chinese](2603.11486.zh.md)

- **Arxiv**: 2603.11486
- **Authors**: Yi Su, Xinchen Luo, Hongtao Cheng, Ziteng Shu, Yunfeng Zhao, Fangyu Zhang, Jiaqiang Liu, Xiao Liang, Yiwu Liu, Ruiming Tang (Kuaishou Inc.)
- **Date**: 2026-03
- **Read date**: 2026-04-22

## Core Contribution
- 首次系统分析生成式推荐模型（OneRec-V2）的量化可行性：其权重/激活分布与 LLM 接近，远比传统推荐模型更容易量化
- 设计了面向 OneRec-V2 的 FP8 PTQ 框架（weights 按 channel 量化，activations 按 token 动态量化），结合 MoE block-wise 量化
- 联合推理基础设施优化（TensorRT 直出、RadixTopK、attention kernel 重设计、MoE Hopper TMA），实现 49% 延迟降低、92% 吞吐提升，在线指标无回退

## Method
量化策略：Linear/Attention/MoE 算子使用 FP8（E4M3），FP32 累积后回 FP16。Weights: per-channel 静态量化；Activations: per-token 动态量化；MoE: 1×128 activation，128×128 weight block-wise 量化。推理侧绕过 ONNX，直接用统一算子库构建 TensorRT 计算图，配合 RadixTopK 和 batch-level 并行 attention kernel。

## Key Results
| 配置 | 延迟 | 吞吐 |
|------|------|------|
| FP16 baseline | 139ms | 205 |
| + Infra 优化 | - | 260 (+27%) |
| + FP8 量化 | - | 346 (+42%) |
| + 算子优化 | 70ms | 394 (+23%) |

在线 A/B（Kuaishou/Kuaishou Lite）核心指标无系统性回退（大部分指标变化 <0.5%）。模型规模：4B 参数，每 token 激活 0.5B。

## Relevance to nanoGenRec
**低相关（工程/部署层面，当前阶段不需要）**：

1. **分布特性验证**：OneRec-V2 的权重/激活分布与 LLM 接近这一发现，间接支持我们的 MoE NTP 模型未来量化的可行性
2. **MoE 量化注意事项**：我们使用 S-tier MoE（6 层，8 专家 top-2）。若未来需要部署加速，block-wise 量化（而非 per-tensor）是 MoE expert 的正确粒度
3. **当前阶段无直接价值**：我们仍处于研究/优化阶段，推理加速不是优先级
4. **未来参考**：当模型规模接近 1B+ 且需要线上部署时，此文章提供了完整的参考路径
