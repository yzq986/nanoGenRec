# MetaGDPO: Alleviating Catastrophic Forgetting with Metacognitive Knowledge through Group Direct Preference Optimization

[English](2511.12113.md) | [中文](2511.12113.zh.md)

- **Arxiv**: 2511.12113
- **Authors**: Lanxue Zhang, Yuqiang Xie, Fang Fang, et al. (IIE-CAS + JIUTIAN)
- **Date**: 2025-11
- **Read date**: 2026-04-22

## Core Contribution
- **GDPO (Group Direct Preference Optimization)**: 从 strong model 离线采样 G=10 responses，用 Bradley-Terry 模型推导排序偏好，优化相邻 pair 而非所有 pair（O(G) vs O(G²)），结合 reference model 约束防止参数漂移
- **MetaKL 数据集**：基于 metacognitive knowledge（模型解题所需认知技能）做数据选择——保留模型不熟悉的复杂题，每类技能保留少量已会的题作 "reminder"，共 5K 样本
- 目标：在 <8B 小模型上做 reasoning 蒸馏，同时防止 catastrophic forgetting

## Method
GDPO 本质：offline DPO 的 group 扩展版，用来近似 GRPO。优势加权 + 相邻 pair 比较：将 G 个 response 按 reward 排序，只优化相邻 (yi, yi+1) pair 的偏好损失，advantage weight Ai = reward_i。证明了 G≥10 时梯度误差 < 10%。Reference model 做 KL 约束（防 drift）。

## Key Results
在 Qwen3-8B / DeepSeek-R1-Qwen-7B / DeepSeek-R1-LLaMA-8B 上：
- METAGDPO vs LIMO/STAR-1/L+S: 数学推理 +5-10% overall AVG，同时保持 general reasoning 和 safety
- L+S (LIMO+STAR-1 混合) 在 Qwen3 上 GSM8K 下降 40%（catastrophic forgetting）；METAGDPO 无此问题
- GDPO: resource-limited 下近似 GRPO 效果（无需在线采样）

## Relevance to nanoGenRec
**低相关——LLM 数学推理蒸馏，与 GR 场景差距较大**：

1. **GDPO 的技术思路值得了解**：作为 offline GRPO 近似（offline 采样 G responses + advantage-weighted pair preference），比标准 DPO 更稳定。但我们的 SP-DPO 问题是离线 preference 已经有了（beam outputs vs target），不需要从 strong model 采样
2. **Catastrophic forgetting 问题存在于 GR RL 阶段**：RL 微调后 NTP 基础能力可能下降（已在 OneLive/SPOT/GenRec 中提到）。MetaKL 的"保留已会知识的 reminder 样本"策略值得借鉴——在 RL 训练集中混入少量高质量 NTP 样本
3. **与 SPOT-BCO 的关联**：两者都关注"只提升 y+ 而不过度抑制 y-"，路径不同（MetaGDPO 靠 reference model KL 约束；SPoT-BCO 靠解耦 BCE 避免梯度消失）
4. **非 GR 论文**：实验都在 NLP reasoning 上，不能直接引用结论证明 GR 适用性
