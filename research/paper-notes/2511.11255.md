# Align3GR: Unified Multi-Level Alignment for LLM-based Generative Recommendation

[English](2511.11255.md) | [Chinese](2511.11255.zh.md)

- **Arxiv**: 2511.11255
- **Authors**: Wencai Ye, Mingjie Sun, Shuhang Chen, et al. (Kuaishou Technology)
- **Date**: 2025-11
- **Read date**: 2026-04-22

## Core Contribution
- **三级对齐框架**：Token-level (SCID 双 tokenization) + Behavior modeling-level (multi-task SFT) + Preference-level (Progressive DPO)
- **Dual SCID Tokenization**：同时为 user 和 item 构建 SCID（SC Encoder = concat(semantic, collaborative) → MLP → RQ-VAE），通过 U2I behavior loss 联合优化 user-item 对齐
- **Progressive DPO (easy→hard)**：SP-DPO（self-play，按 prefix-ngram overlap 分 easy/medium/hard 三阶段）+ RF-DPO（real feedback：disliked→neutral→liked 三级）；使用 Softmax-DPO（多 rejected 变体）
- 全流程部署于工业广告推荐平台，A/B test +1.432% revenue

## Method
Tokenizer: semantic encoder (T5) + collaborative encoder (DIN) → concat → SC Encoder (MLP) → U2I InfoNCE loss 对齐 → RQ-VAE 3层×256 量化 → SCID。SFT: 5类 task（Sequential item prediction + 4 alignment tasks），user SCID 注入所有 prompt。Progressive DPO: 先 SP-DPO（self-play，3阶段），再 RF-DPO（real feedback，2阶段），每阶段 policy model → 下阶段 reference model。

## Key Results
| Dataset | Baseline (EAGER-LLM) | Align3GR | Improvement |
|---------|---------------------|----------|-------------|
| Instruments | R@10=0.1224 | 0.1442 | +17.8% |
| Beauty | R@10=0.0830 | 0.0994 | +19.8% |
| Yelp | R@10=0.0569 | 0.0679 | +19.3% |

Ablation: 最大增益来自 Preference-level alignment (SP-DPO + RF-DPO)。Dual SCID 比 item-only +0.9pp R@10，CF+U-I alignment 再 +0.5pp。

## Relevance to nanoGenRec
**高相关——与我们架构最接近的 Kuaishou 实践，SP-DPO 改进路径值得参考**：

1. **SP-DPO Easy→Hard 分级**：我们的 SP-DPO 目前是单阶段。Align3GR 的 prefix-ngram overlap 分级思路值得借鉴——先用完全不重叠的 (easy) pairs 训练基础能力，再用高重叠 (hard) pairs 精调细粒度区分能力。这与 SPOT 论文的 LCS 质量过滤方向一致（都关注 prefix/LCS overlap）
2. **RF-DPO（真实用户反馈）**：online A/B → dislike/neutral/like 三级构造 preference pairs。这是 SP-DPO 之后的自然延伸，我们当前完全靠 self-play。但实施需要在线系统支持
3. **Dual SCID 的增益较小**：相比 preference-level alignment，tokenizer 改进（+0.9-1.4pp）相对次要。这与我们的判断一致：先把 NTP/RL 做好，tokenizer 改进放后面
4. **Softmax-DPO 多 rejected 变体**：使用多 rejected（20个）的 Softmax-DPO，本质上提供了更丰富的 negative 信号。与 GRPO 的 group-wise advantage 方向有共同点
5. **SCID 4 tokens/item**：用了 <a><b><c><d> 4 token，比我们的 3 token 多一层；RQ-VAE 3层×256 codebook
