# OneLive: Dynamically Unified Generative Framework for Live-Streaming Recommendation

[English](2602.08612.md) | [Chinese](2602.08612.zh.md)

- **Arxiv**: 2602.08612
- **Authors**: Shen Wang, Yusheng Huang, Ruochen Yang, et al. (Kuaishou Technology)
- **Date**: 2026-02
- **Read date**: 2026-04-22

## Core Contribution
- **Dynamic Tokenizer (IA Embedding)**: 实时融合直播内容（MLLM 30s 滑窗）+ 协同信号（dual-tower post-alignment），通过 Res-Kmeans (8192×3) 量化为动态 SID，解决直播内容持续变化问题
- **Sequential MTP + QK Norm**: 3 个串联 decoder（主+2个轻量），共享 KV cache；QK Norm 防止深层训练中 logit explosion，实现 62% QPS 提升（6层）同时稳定训练
- **Unified Multi-Objective RL**: GRPO 在多目标对齐（CTR/LVTR/GTR/WTR）上显著优于 DPO，部署在快手直播，400M 日活

## Method
**Dynamic Tokenizer**: MLLM（30s 窗口）生成 content embedding，dual-tower 做 user-author alignment（InfoNCE）得 IA Embedding，Res-Kmeans 3层×8192 量化 → 动态 SID（每30s更新一次）。

**Architecture**: Decoder-only（lazy decoder：user 历史序列直接做 KV context，不编码），Time-Aware Gated Attention（multi-granular time bias + learnable sigmoid gate），3层 MTP 共享第一层 Attention KV cache。

**RL**: Reference model 周期同步参数，采样候选，ranking model (Pantheon) 打分，GRPO 优化。仅 1% query 做 RL 训练。

## Relevance to nanoGenRec
**中等相关——直播场景特殊性限制直接应用，但多个组件值得参考**：

1. **QK Norm**：我们的 S-tier MoE 也用 bf16 训练，若扩展到更深模型时遇到训练不稳定（loss spike），可以直接引入 QK Norm（2行代码）。目前 6 层未出现问题，但值得关注
2. **Sequential MTP**：解决 beam search 推理瓶颈——推理时 q₀ 的 KV cache 被 q₁/q₂ 共享，+62% QPS。我们 3-token SID 结构完全相同，是推理加速的可行路径（不影响训练质量）
3. **GRPO > DPO in RL**：独立验证了 OneMall 的结论。GRPO 的组内 advantage 归一化提供更丰富梯度信号，优于 pairwise DPO。强化了我们探索 GRPO 的理由
4. **动态 Tokenizer**：仅在直播场景必要（内容每30s变化）。我们的商品/短视频 SID 是静态的，不需要
