# FlexCode: Semantics Meet Signals — Dual Codebook Representation Learning for Generative Recommendation

[English](2511.20673.md) | [中文](2511.20673.zh.md)

- **Arxiv**: 2511.20673
- **Authors**: Zheng Hui, Xiaokai Wei, et al. (Roblox + UMD + UIC + Cambridge)
- **Date**: 2025-11
- **Read date**: 2026-04-22

## Core Contribution
- **FlexCode**: 双 codebook 框架——语义 codebook (CSEM, RQ-VAE on text embedding) + 协同 codebook (CCF, RQ-VAE on SASRec embedding)，用 popularity-aware MoE router 动态分配 token budget
- **核心问题**：单 codebook 强迫语义信号和协同信号共享同一表示空间，导致 head items（交互丰富）和 tail items（冷启动）都受损
- **Cross-Codebook Alignment (CCA)**：InfoNCE 对齐两个 codebook 的重建向量，防止表示空间漂离，同时缓解 codebook collapse
- **Popularity-Aware Token Allocation (PATA)**：轻量 MLP 路由，输入 [log(1+freq), age, sparsity, uncertainty]，输出各 codebook token 数量

## Method
双 codebook 分别用 RQ-VAE 学习：LSCL（语义重建）+ LCCL（协同重建）+ LCCA（跨 codebook InfoNCE）+ LARG（autoregressive generation）+ Llb（负载均衡）+ Lsmooth（分配平滑性）。训练目标全部联合优化。推理时将 αi 取整离散化。

## Key Results
公开数据集（Beauty/Sports/KuaiRand）：
- FlexCode vs URL (best baseline): Beauty R@10: 0.0769 vs 0.0736 (+4.5%); Sports: 0.0471 vs 0.0449 (+4.9%); KuaiRand NDCG@10: 0.0632 vs 0.0585 (+8.0%)
- FlexCode-Fix (50/50 固定 split) 已超过所有 baseline，MoE 动态分配再额外 +12.5% on KuaiRand

工业数据集（1.5M users, 1M items, 45M interactions）：
- vs SASRec baseline: +13.2% NDCG@10, +16.5% HR@10
- Head items: +3.0% NDCG@10（CF tokens 精确 memorization）
- Tail items: +11.3% NDCG@10（Semantic tokens 语义泛化），最大优势

## Relevance to gr_demo
**中等相关——双 codebook 思路有价值，但与当前 MLP-FSQ 改造路径不同**：

1. **与 ResKmeansFSQ 互补的视角**：OneMall 从"冲突率"角度改进 tokenizer，FlexCode 从"语义 vs 协同表示分离"角度改进。两者都指向同一问题：单一 codebook 质量不足。我们 MLP-FSQ 是纯语义，无协同信号
2. **CF Codebook 思路**：引入额外 SASRec embedding 作为协同 codebook 可能有效，但需要额外的 CF tower 训练（同 TCA4Rec 问题）。目前 overhead 较大
3. **PATA token budget 分配**：popularity-based 动态 token 数量是一个有意思的扩展方向。但我们当前 3-token 是 fixed，需要重新设计 decoder 才能使用变长 SID
4. **CCA 防 codebook collapse**：跨 codebook InfoNCE 对齐重建向量，这一技巧可以借鉴到 Res-Kmeans 多层 codebook 的对齐训练中
