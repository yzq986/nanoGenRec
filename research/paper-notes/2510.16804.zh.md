# LAC: The Layout Is the Model — On Action-Item Coupling in Generative Recommendation

[English](2510.16804.md) | [中文](2510.16804.zh.md)

- **Arxiv**: 2510.16804
- **Authors**: Xiaokai Wei, Jiajun Wu, Daiyao Yi, Reza Shirkavand, Michelle Gong (Roblox + UMD)
- **Date**: 2025-10
- **Read date**: 2026-04-22

## Core Contribution
- **Token Layout as First-Class Design Axis**：GR 中 item/action token 的排布和可见性策略（layout）是独立设计维度，与 backbone 和数据同等重要
- **三原则**：P1 最大化 input/output 有用信息；P2 保留 "action given item" 的因果方向；P3 无信息泄漏
- **Lagged Action Conditioning (LAC)**：(I_t, A_{t-1}) → (I_{t+1}, A_t)，非 interleaved，序列长度 L=T（不翻倍），满足所有三原则。比较对象：HSTU-style (interleaved, 2T) 和 PC (patched conditioning, late fusion)
- LAC vs Interleaved：-31% 到 -43% 训练成本，同等或更好的 retrieval + ranking 准确率

## Method
LAC 在推理时支持 parallel scoring：将 history + C candidates pack 进一个序列，block-causal mask（candidates 各自 attend 所有 history），一次 forward 完成所有 candidate 的 action 预测。LAC 的 "mismatch" 是有意为之——Transformer 自动学习 lag-by-one pattern（通过注意力 attend I_t-1 位置聚合该位置的 A_t-1 信息）。

实验：>10 种 layout 组合，12层 GPT-2 style 85M → 355M，Amazon Beauty / Kuaisar / 工业数据集。

## Key Results
工业数据集（50M, 355M, 774M 参数）：
- LAC HR@1: 0.2516 vs Interleaved: 0.2511 (+0.2%)，但 Duration RMSE: 0.1973 vs 0.1985 (-0.6%)
- Training cost: LAC -31.3% (774M) to -42.7% (355M)
- PC (non-interleaved late fusion): HR@1=0.2487，Duration RMSE=0.2164 — 明显差于 LAC 和 Interleaved

## Relevance to nanoGenRec
**中等相关——我们的 action embedding 设计与 LAC 框架有直接对应**：

1. **我们的 action token 与 LAC 的关系**：我们的 IDEA-feat-1 (action embedding) 目前是将 action 直接拼入 item token 的 input（`[item_emb; action_emb]`，类似 LAC 的 non-interleaved 形式）。按照 LAC 的 P2 原则，`(I_t, A_{t-1}) → (I_{t+1}, A_t)` 是更好的 layout：用上一步的 action 作为条件，同时预测下一步 item 和当前 action
2. **LAC 是 EXP-025 beam_passes 的理论支撑**：EXP-025 中 beam search 传递 time_gap + action 特征是在 inference 时的 "action carry-forward"。LAC 正式化了这一思路：action 信号应该 lag 一步作为下一步 item 的 conditioning
3. **Parallel scoring 对未来 ranking 阶段有价值**：当我们 GR 模型从 retrieval 扩展到 ranking（预测 action 如 watch time、purchase），LAC 的 parallel scoring 方式（pack history + C candidates，block-causal mask）是 O(T+C) inference，远优于逐 candidate 方式
4. **PC 明显差于 LAC**：晚期融合（late fusion）的 action conditioning 不如早期融合（early fusion via attention）。我们如果要支持 action prediction，应优先考虑 LAC 而非 PC
