# Semantic ID Next Token Prediction (NTP) Metric

[English](SID_PREDICTION.md) | [Chinese](SID_PREDICTION.zh.md)

## Overview

Semantic ID prediction evaluation metric implemented based on the OneRec paper (arxiv 2506.13695).

**Core idea**: Use the Semantic ID tokens of k items that the user has interacted with in history to predict the Semantic ID of the next item.

---

## Data Design

### 1. Original data

User behavior data exported from `export_user_behavior.py`:

```
┌──────────┬──────────┬───────────────┬────────────┐
│ uid      │ iid      │ action_bitmap │ first_ts   │
├──────────┼──────────┼───────────────┼────────────┤
│ user_A   │ item_1   │ 3             │ 1711800000 │
│ user_A   │ item_2   │ 1             │ 1711800100 │
│ user_B   │ item_5   │ 7             │ 1711800050 │
│ ...      │ ...      │ ...           │ ...        │
└──────────┴──────────┴───────────────┴────────────┘
```

- `action_bitmap > 0`: positive behavior (click, like, share, etc.)
- `first_ts`: first interaction timestamp

### 2. Semantic ID format

3-layer Semantic ID generated by RKMeans:

```
"12_34_56" → [12, 34, 56]  # Layer 1, Layer 2, Layer 3
```

256 clusters per tier (configurable).

### 3. Sample construction

**Sliding window** (n_items=10):

```
User sequence: [item_1, item_2, ..., item_15] (sorted by time)

Sample 1: input = [item_1 ~ item_10], target = item_11
Sample 2: input = [item_2 ~ item_11], target = item_12
Sample 3: input = [item_3 ~ item_12], target = item_13
...
```

**Token format**:

```
input_tokens:  [L1,L2,L3, L1,L2,L3, ..., L1,L2,L3]  # 10 items × 3 layers = 30 tokens
                item_1    item_2        item_10

target_tokens: [L1, L2, L3] # 3 tokens for the next item
target_ts: 1711800500 # Used for global time sorting
```

### 4. Time sorting & Split

**Key**: Avoid time penetration (data leakage)

```
All samples are sorted globally by target_ts:

Timeline:
|<---------- Train (80%) ---------->|<--- Eval (20%) --->|
Sample_1 Sample_2 ... Sample_M Sample_M+1 ... Sample_N
T_start T_split T_end
```

- **Train**: 80% of the earlier samples, trained after shuffle
- **Eval**: 20% of samples later in time, only evaluated but not trained

## Model architecture

### AutoregressiveNTPModel

```
Input: 30 tokens (10 items × 3 layers)
       ↓
Token Embedding + Position Embedding
       ↓
Transformer Decoder (Causal Self-Attention) × 2 layers
       ↓
Output Projection → 256 classes

Generation (autoregressive):
  Step 1: [30 tokens] → predict L1
  Step 2: [30 tokens, L1] → predict L2
  Step 3: [30 tokens, L1, L2] → predict L3
```

**Parameters**:
- `embed_dim`: 64
- `n_heads`: 4
- `n_transformer_layers`: 2
- Total: ~170K params

### Beam Search

Use Beam Search (beam_size=5) when inferring:

```
P(item) = P(L1) × P(L2|L1) × P(L3|L1,L2)
```

## Evaluation indicators

| Metric | Definition | Meaning |
|------|------|------|
| **Perplexity** (main Metric) | exp(avg_loss / n_layers) | The lower the better, random baseline = 256 |
| **All Correct** | Proportions of all 3 layers | Beam Search Output |
| **Layer Acc** | Individual accuracy for each layer | Beam Search Output |
| **Hit@K** | Proportion of correct answers in top-K Medium | Calculated under Teacher Forcing |

### Quality Judgment

```python
thresholds = {
    'excellent': 50,   # PPL <= 50
    'good': 100,       # PPL <= 100
    'acceptable': 150, # PPL <= 150
    'poor': > 150
}
```

Random baseline PPL = 256 (n_clusters)

## How to use

### Command line

```bash
# Run only SID Prediction
python run.py eval-all --only-sid --models qwen3-0.6b

#Adjust parameters
python run.py eval-all --only-sid \
    --models qwen3-0.6b \
    --sid_sample_users 50000 #Number of sample users
```

### Code call

```python
from metrics import SemanticIDPredictionMetric

metric = SemanticIDPredictionMetric()
result = metric.compute(
    embeddings=embeddings,
    semantic_ids=semantic_ids,
    behavior_data=behavior_data,
    content_id_to_idx=content_id_to_idx,
    n_items=10, # Number of historical items
    beam_size=5, # Beam Search width
    sample_users=50000,
    device='cuda',
)

print(f"Perplexity: {result.value}")
print(f"Details: {result.details}")
```

## Design considerations

### Why Split by time?

| Solution | Problem |
|------|------|
| Random split user | The sequence of Different users may have interleaved Time and may be leaked |
| Intra-user split | No cross-user learning, but penetration may still occur during scrolling training |
| **Global Time split** | Strictly ensure that eval is after train and there is no leakage |

### Why use Perplexity?

- OneRec paper uses Cross-Entropy Loss / Perplexity
- Accuracy is not suitable for recommended scenarios (too strict)
- Perplexity interpretable: the "perplexity" of the model, random = n_clusters

### Why 1 Epoch?

- Under strict time split, multiple epochs overfit the train data.
- Single epoch is closer to the online learning scenario
- Experiments show that 1 epoch is sufficient for convergence
