## EXP-036: Clean Features NTP — From-Scratch Training with time_gap + action_level

**Date**: 2026-04-28
**Status**: completed
**Results**: experiments/ntp_checkpoints/exp036-no-features/ + experiments/ntp_checkpoints/exp036-full-features/

### Background

exp025（R@500=63.6%）是目前唯一带 features 的 NTP checkpoint，但它不是干净的对照实验：
- 基于 exp023 做了 beam-passes SFT（而非从头训练）
- 与 exp020（R@500=66.2%）的差异不只是 features，还有训练方式、数据集不同

EXP-023 的 "all features" config（time_gap + action + segment）R@500 只有 55.0%，比 segment_only（61.2%）还低，主要原因是**训练-推理信息泄漏**（EXP-024 已分析）。EXP-025 修复了泄漏（delayed features + beam_passes），但只做了 1 epoch beam_passes，训练不充分。

**目标**：用与 exp020 完全相同的训练条件（相同数据集、相同超参、从头训练），唯一变量是是否加 features，得到干净的对照结果。

### Hypothesis

features（time_gap + action_level + segment_emb）在训练-推理 gap 修复后，从头训练应能超越 exp020：

| 指标 | exp020 (no features) | EXP-036 (features, 从头训) | 预期变化 | 理由 |
|------|---------------------|--------------------------|---------|------|
| PPL | 16.3 | 预期 ≤16.3 | ↓ | features 提供额外区分信号 |
| R@10 | 14.1% | 预期 ≥14.1% | ↑ | time_gap 区分时效性 |
| R@500 | 66.2% | 预期 ≥67% | ↑ | action_level 区分交互强度 |
| 训练时间 | ~62min | ~65min | ↑小 | features embedding 计算量微增 |

### Design
- **Variable**: features on/off（Config A: 无 features 复现 exp020；Config B: time_gap + action_level + segment）
- **Fixed**: 相同数据集（exp023-14d-features，已含 time_gaps/action_levels）、相同超参（lr=1e-3, batch=4096, 1 epoch, s-tier model）、相同 SID cache（exp013-4096x3-12d-binary）
- **Metric**: PPL, R@10, R@500（full eval n_recall=1000）；新增 kl_mean 作为后续 RL 基准
- **Data**: experiments/ntp_data/exp023-14d-features（已有，无需重新 preprocess）

### Run
`bash experiments/scripts/exp-036.sh`

### Results

| 指标 | Config A (no features) | Config B (full features) | exp020 基线 | Δ(B-A) |
|------|----------------------|------------------------|------------|--------|
| R@10 | 9.4% | **10.9%** | 14.1% | +1.5pp ✅ |
| R@500 | 55.3% | **59.0%** | 66.2% | +3.7pp ✅ |
| PPL | 34.9 | **27.3** | 16.3 | ↓7.6 ✅ |
| train_loss | 3.620 | **3.507** | — | -0.113 ✅ |
| 训练时长 | 7min50s | 7min58s | ~62min | 相近 |

> 注：Config A/B 绝对值均低于 exp020（66.2%），因为使用 exp023-14d-features 数据集（item 集合不同），与 exp020 使用的 exp016-14d 数据不可直接对比。**关键结论看 B-A 差值**。

### Analysis

**假设验证结果**：

| 假设 | 预期 | 实际 | 结论 |
|------|------|------|------|
| PPL: B < A | ↓ | 27.3 vs 34.9 (↓7.6) | ✅ 验证 |
| R@10: B > A | ↑ | 10.9% vs 9.4% (+1.5pp) | ✅ 验证 |
| R@500: B > A | ↑ | 59.0% vs 55.3% (+3.7pp) | ✅ 验证 |

**关键结论**：

1. **features 有效**：相同数据、相同超参、唯一差异是 features on/off，Config B 全面优于 Config A。PPL 从 34.9 降到 27.3（降幅 22%），R@500 提升 3.7pp，效果显著。

2. **绝对值低于 exp020 的原因**：exp023-14d-features 数据集与 exp016-14d 是不同的 item 集合/时间窗口，不可直接跨数据集比较。该实验的目的是控制变量验证 features 本身的效果，这个目的已达到。

3. **exp025 的劣势已解释**：exp025（beam-passes SFT，R@500=63.6%）比 exp020 差，是因为基于 exp023（segment-only, 61.2%）做 beam-passes，训练起点本身就弱。Config B 从头训练避免了这个问题。

4. **新 SFT 起点**：exp036-full-features 是目前唯一从头训练、训推一致、features 有效的 NTP checkpoint，可作为下一轮 RL 的 SFT 起点。

### Next Steps

**完整 features 路线链路（对标 exp016→017→019/020→029）：**

```
exp036-B (NTP + features)
  → EXP-037: SP-DPO Easy+Medium（prefix-locked beam，基于 exp036-B 生成 pairs）
    → EXP-038: RF-DPO Joint NTP+DPO λ=0.03（ref = SP-DPO checkpoint，对标 exp019/020）
      → EXP-039: ECPO on-policy（对标 exp029）
```

**重要背景（防止再次理解错误）**：
- exp020（SFT SOTA）的完整来源：`exp016(NTP) → exp017(SP-DPO) → exp019/020(RF-DPO, ref=SP-DPO)`
- RF-DPO **不是**直接在 NTP 上做的，ref model 是 SP-DPO checkpoint（exp017 fixed-medium）
- Pure RF-DPO 会灾难遗忘（exp018 PPL 爆炸到 50K+），必须 Joint NTP+DPO 才能稳定
- SP-DPO pairs 必须用当前 checkpoint（exp036-B）重新生成，不能复用 exp017 的 pairs（beam candidates 依赖模型分布）
- RF-DPO pairs（真实用户反馈）可以复用 exp018 的数据（行为数据与模型无关）
